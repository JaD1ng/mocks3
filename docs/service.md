# 微服务业务流程

## 上传流程

### 1. 客户端上传请求
```
PUT /{bucket}/{key}
```

### 2. 服务调用链
1. **API网关** (Nginx) → 路由请求到 S3 API 服务
2. **S3 API服务** → 业务编排，调用以下服务：
   - 调用**元数据管理服务**检查对象是否存在
   - 调用**分布式存储服务**写入文件到多个存储节点
   - 调用**元数据管理服务**保存文件元数据

### 3. 存储逻辑
- **分布式存储服务**顺序写入 stg1、stg2、stg3 三个存储节点
- 三个节点都成功才认为上传成功
- 生成 MD5 哈希校验文件完整性

### 4. 元数据管理
- **元数据管理服务**将文件信息存储到 PostgreSQL
- 缓存热点元数据到 Redis
- 记录存储节点列表、文件大小、创建时间等

## 下载流程

### 1. 客户端下载请求
```
GET /{bucket}/{key}
```

### 2. 服务调用链
1. **API网关** → 路由到 S3 API 服务
2. **S3 API服务** → 业务编排：
   - 调用**元数据管理服务**查询文件元数据
   - 调用**分布式存储服务**读取文件内容
   - 如果文件不存在，尝试从第三方服务获取

### 3. 数据读取策略
- **优先从 stg1 节点读取**
- 如果 stg1 失败，尝试其他存储节点
- 如果所有节点都失败，调用第三方服务获取数据
- 从第三方获取成功后，异步写入本地存储节点

### 4. 缓存机制
- **元数据管理服务**优先从 Redis 缓存查询元数据
- 缓存未命中时查询 PostgreSQL 数据库
- 热点文件元数据自动缓存

### 5. 响应处理
- 设置正确的 HTTP 头：Content-Type, Content-Length, ETag 等
- 返回文件二进制数据

## 查询流程

### 1. 列出对象请求
```
GET /{bucket}
```

### 2. 元数据查询请求
```
HEAD /{bucket}/{key}
GET /api/v1/objects
GET /api/v1/stats
```

### 3. 服务调用链
1. **API网关** → 路由到相应服务
2. **S3 API服务** 或 **Admin API服务** → 调用**元数据管理服务**
3. **元数据管理服务** → 查询缓存或数据库

### 4. 查询类型
- **对象列表查询**: 分页返回 bucket 中的对象列表
- **对象元数据查询**: 返回单个对象的详细信息
- **统计信息查询**: 返回系统整体统计数据
- **搜索查询**: 基于关键词搜索对象

### 5. 读写分离
- **读操作**: 优先从 Redis 缓存读取，缓存未命中时查询 PostgreSQL 从库
- **写操作**: 直接操作 PostgreSQL 主库，异步更新缓存

---

## 删除流程

### 1. 客户端删除请求
```
DELETE /{bucket}/{key}
```

### 2. 服务调用链
1. **API网关** → 路由到 S3 API 服务
2. **S3 API服务** → 业务编排：
   - 调用**元数据管理服务**删除元数据记录
   - 发送异步删除任务到**异步任务处理服务**

### 3. 异步删除
- **异步任务处理服务**处理文件物理删除任务
- 调用**分布式存储服务**从各个存储节点删除文件
- 清理相关缓存数据

### 4. 事务保证
- 先删除元数据，后异步删除物理文件
- 如果物理删除失败，通过定时任务重试清理

---

## 错误处理和监控

### 1. 错误注入服务
- 模拟网络层故障、服务层故障、存储层故障、资源层故障、业务层故障
- 可配置故障注入规则和概率
- 支持针对特定服务或接口的故障模拟

### 2. 监控和观测
- **Prometheus** 收集各服务的性能指标
- **OpenTelemetry** 记录分布式链路追踪
- **ElasticSearch** 存储日志和链路数据
- 关键指标：请求 QPS、响应时间、错误率、存储使用率

### 3. 服务发现
- **Consul** 管理服务注册和发现
- 服务健康检查和自动故障转移
- 动态配置管理通过 Consul KV
